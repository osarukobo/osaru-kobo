# pdf-edit スキル作成の記録

## 要点

- `/pdf-edit` スキルで発注書PDFへの会社情報・日付・ハンコ挿入を自動化できるようにした
- チャット側での試行錯誤（武蔵浦和、狛江ちとせ保育園）から得た技術知見をスキルに集約
- `~/.claude/skills/pdf-edit/SKILL.md` に配置（グローバルスキル）

## 背景

発注書PDFが不定期で届くたびに、左下の枠内に以下を手作業で入力していた：

- 発注日（PDF右上の「発注日」フィールドの値）
- 会社名・住所・代表者名（固定：株式会社やどり木）
- 角印（PNG画像）

チャットでClaudeに依頼すると毎回試行錯誤で時間がかかっていたため、スキル化した。

## 技術的な知見（チャットの試行錯誤から）

### 採用した方式

`reportlab`（オーバーレイ作成）+ `pypdf`（merge_page合成）

### 失敗した方式

- `pymupdf`単体での貼り付け：座標系の変換行列問題で不安定
- ページをラスタライズして再合成：枠線・元テキストが消える

### 主なハマりポイント

1. **透過PNGのバウンディングボックスずれ**
   - reportlabで透過PNGを使うとサイズが一定しない
   - 対処：Pillowで白背景PNG変換してから使う

2. **merge_page時のスケールずれ**
   - PDFによってはcm変換行列が埋め込まれていてスケールがずれる
   - 対処：仮配置→PyMuPDFで実測→差分補正（1回だけ）

3. **ハンコのサイズ感**
   - 枠の高さ×0.72が自然なサイズ比率（河崎様邸PDFから計測）
   - 上端から5pt、右端から5pt内側に配置

### 必要なライブラリ

```bash
pip install reportlab pypdf pymupdf pillow
```

## 使い方

発注書PDFとハンコ画像を渡すだけでよい。

```
PDF届いた → /pdf-edit スキルを呼ぶ → PDFパスとハンコ画像を渡す → 完成PDF出力
```
